## SBC4

- `Tease:` FetchMoji chooses emoji from meaning, not official labels.
- `Lede:` The live product's next iteration connects reliable local search, useful intent pages, extension reuse, and accurate privacy claims.
- `Why it matters:` Growth only compounds when discovery leads to a trustworthy choice and a reason to return.
- `Go deeper:` Review the target surfaces below, then read How it works.

FetchMoji is a natural-language emoji finder. A person types the phrase they
would actually say—such as “I am proud of you,” “that was awkward,” or “I am
overthinking this”—and gets a short ranked set of emojis to copy.

Most emoji pickers assume the person already knows an official emoji name.
FetchMoji starts from intent instead. It distinguishes nearby tones, explains
why a candidate fits, and lets the person move from uncertainty to a usable
choice without sending the query to a search server.

The live product already proves the core search. This spec defines its next
growth iteration:

- the homepage explains and demonstrates the product in crawlable HTML;
- search never loses a query while the local index initializes;
- intent pages answer real phrase decisions with reviewed, non-commodity value;
- a low-permission browser extension puts search inside the writing workflow;
- measurement proves acquisition and activation without collecting query text;
- search and browser agents can understand the public site through ordinary,
  semantic web foundations.

The target is not “more pages” or “more launch-day attention.” It is a product
that earns discovery, answers the intent well, and becomes easy to reuse.
